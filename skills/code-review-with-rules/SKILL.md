  # 代码审查规范

  ## 必须检查的项目

  ### 1. 代码质量
  - 代码可读性和可维护性
  - 命名规范（变量、函数、类名）
  - 代码复杂度（避免过深嵌套）
  - 重复代码检查

  ### 2. 安全性检查
  - 输入验证
  - SQL 注入防护
  - XSS 防护
  - 敏感信息泄露（密码、密钥等）

  ### 3. 性能考虑
  - 不必要的循环
  - 数据库查询优化
  - 内存泄漏风险

  ### 4. 团队规范
  - 符合项目代码风格
  - 注释是否完善
  - 错误处理是否健全

  ## 审查输出格式

  审查完成后，按以下格式输出：

  **✅ 通过的检查项**
  - [列出通过的项目]

  **⚠️  需要改进的地方**
  - [具体问题描述]
  - [文件位置: 行号]
  - [建议修复方案]

  **❌ 严重问题**
  - [关键问题描述]
  - [必须修复]

  **📊 总体评分**: [1-10分]
  **建议**: [总体建议]
